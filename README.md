# EAS Scalar Fields Modeling — Framework Release Guard

This repository directory set publishes the current canonical `enforceable-rank3-modeling` framework release for runner version checks.

The important release files are in:

```text
releases/current/
  enforceable_rank3_modeling.zip
  FRAMEWORK_RELEASE_MANIFEST.json
  FRAMEWORK_RELEASE_MANIFEST.sig          # generated by you before enforcement use
  FRAMEWORK_RELEASE_PUBLIC_KEY.pem        # generated by you before enforcement use
  SHA256SUMS.txt
```

The framework release guard should trust the signed release manifest, not a version string alone. Candidate/admission runs should refuse to execute unless the locally installed framework hash equals `latest_framework_sha256` in the signed manifest and all required capabilities are present.

## One-time release signing setup

Generate a release signing key locally. Do **not** commit the private key.

```bash
python tools/generate_release_key.py \
  ~/.rank3/release_private_key.pem \
  releases/current/FRAMEWORK_RELEASE_PUBLIC_KEY.pem
```

Sign the current release manifest:

```bash
python tools/sign_release_manifest.py \
  releases/current/FRAMEWORK_RELEASE_MANIFEST.json \
  ~/.rank3/release_private_key.pem \
  releases/current/FRAMEWORK_RELEASE_MANIFEST.sig
```

Verify before committing:

```bash
python tools/verify_release_manifest.py \
  releases/current/FRAMEWORK_RELEASE_MANIFEST.json \
  releases/current/FRAMEWORK_RELEASE_MANIFEST.sig \
  releases/current/FRAMEWORK_RELEASE_PUBLIC_KEY.pem \
  releases/current/enforceable_rank3_modeling.zip
```

Commit these public files:

```text
releases/current/enforceable_rank3_modeling.zip
releases/current/FRAMEWORK_RELEASE_MANIFEST.json
releases/current/FRAMEWORK_RELEASE_MANIFEST.sig
releases/current/FRAMEWORK_RELEASE_PUBLIC_KEY.pem
releases/current/SHA256SUMS.txt
tools/*.py
docs/*.md
```

Do **not** commit:

```text
~/.rank3/release_private_key.pem
any private signing key
any private encryption key
```

## Policy

- Candidate/admission runs must use the latest signed manifest and exact framework hash.
- Archive reproduction runs may use old framework versions, but must be labeled `archive_reproduction` and may not be admitted as current evidence.
- Operators should pass configuration suites only, not framework ZIPs, scripts, or executable runners.
