# GitHub Install and Publish v0.1.44

Release label: `0.1.44-field0-locked-admissibility-simulation`

## Install from a tagged release

```bash
python -m pip install "git+https://github.com/sheepdoggie/EASScalarFieldsModeling.git@v0.1.44#egg=enforceable-rank3-modeling"
```

## Run the field-0 locked simulation

```bash
rank3-run-field0-locked-simulation \
  --output v0144_field0_locked_run \
  --cycles 6 \
  --point-count 6
```

To run one selected pre-run admissibility spec:

```bash
rank3-run-field0-locked-simulation \
  --output v0144_split_admissibility_run \
  --spec LOCKED_CONJUGATE_SPLIT_ON_FIRST_ASSOCIATION \
  --cycles 6 \
  --point-count 6
```

## Write approval packet

```bash
rank3-write-field0-locked-simulation-packet --output v0144_field0_locked_approval_packet
```

## Verify local release identity

```bash
rank3-check-release-identity \
  --repo-root . \
  --manifest releases/current/FRAMEWORK_RELEASE_MANIFEST.json \
  --framework-zip releases/current/enforceable_rank3_modeling_v0.1.44_field0_locked_admissibility_simulation.zip \
  --framework-tar-gz releases/current/enforceable_rank3_modeling_v0.1.44_field0_locked_admissibility_simulation.tar.gz
```

## Operational constraints

A run may impose field-0 scalar point values and associations. It may also select a frozen admissibility spec before execution. After field 0, all fields must be generated by SOO plus that locked admissibility spec. No structure labels, endpoint classes, target tuples, slot-specific sign policies, or runtime admissibility changes are generator inputs.
