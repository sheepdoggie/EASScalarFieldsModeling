# TDD: v0.1.35 Gradient-Governed Vacuum-Split Path Accommodation

## Status

`READY FOR CERTIFICATION PLAN: yes`

`READY FOR THEOREM CERTIFICATION: no`

This release prepares a frozen theorem/admissibility/control package for the next certification planning step. It does not certify a theorem and it does not execute a theorem-certification model run.

## Narrow theorem target

The theorem target is not electric charge, Coulomb behavior, QED, leptons, or empirical attraction/repulsion. The target is the scalar-field/path-sector statement:

```text
Gradient-Governed Scalar-Sign Path-Accommodation Theorem

Given:
1. vacuum points are undefined until first association,
2. first association resolves a vacuum point into equal/opposite scalar branches,
3. each branch carries three association slots,
4. new associations minimize scalar-gradient burden,
5. new association proposals commit after one cycle,
6. relational paths require determinate scalar gradient,
7. sign-coherent triangle/shell supports have formed without imposed support labels,

then, as a theorem-facing proof obligation:
    opposite scalar-sign support pairs produce center no-gradient invalidity
    and an admitted topology policy removes the invalid carrier;

    same scalar-sign support pairs produce center ambiguous-gradient invalidity
    and an admitted topology policy inserts a vacuum point.
```

## Admissibilities versus hypotheses

The following items are admissibility candidates frozen for operator review:

- A1 undefined-vacuum association state.
- A2 first-association vacuum split into equal/opposite scalar branches.
- A3 six branch association slots per split vacuum site.
- A4 gradient-minimizing new association selection.
- A5 one-cycle association latency.
- A6 determinate-gradient requirement for relational path carriers.
- A7 no-gradient and ambiguous-gradient centers cannot persist as relational path carriers.

The following items are not primitive admissibilities in this release:

- H1 opposite-sign support pairs produce center no-gradient invalidity.
- H2 same-sign support pairs produce center ambiguous-gradient invalidity.
- H3 no-gradient invalidity is accommodated by removal.
- H4 ambiguous-gradient invalidity is accommodated by vacuum insertion.
- H5 dangling/vacuum reassociation follows admitted transactions.

Those items are proof obligations and transaction-policy obligations. If they are implemented as primitive dispatch from sign labels, support labels, or target `Delta L`, certification must be blocked.

## Generator/readout separation

The generator may receive only scalar signs, scalar values, undefined-vacuum state, association candidates, frozen admissibility rules, and a cost functional identifier.

The generator must not receive same/opposite labels, charge labels, handedness labels, target `Delta L`, expected `Delta L`, predeclared support membership, predeclared center action, or preselected path outcome.
